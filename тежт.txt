Работата с GitHub обикновено включва използването на Git (за управление на версиите локално) и понякога на GitHub CLI (за директно управление на функциите на GitHub от терминала). 
Ето най-важните команди, разделени по категории:
1. Основни Git команди (Локален работен процес) 
Тези команди се използват за проследяване на промените във вашия проект.
git init – Инициализира ново локално хранилище (репозитори) в текущата папка.
git clone <url> – Копира съществуващо отдалечено хранилище на вашия компютър.
git status – Показва състоянието на файловете (кои са променени, кои са подготвени за запис).
git add <file> – Добавя конкретен файл към "staging" зоната (подготовка за запис).
git add . – Добавя всички променени файлове.
git commit -m "съобщение" – Записва подготвените промени в историята с кратко описание.
git log – Показва хронологията на всички записи (commits).

2. Синхронизация с GitHub (Remote)
Използват се за качване и сваляне на промени от GitHub. 
git push – Качва вашите локални записи в отдалеченото хранилище в GitHub.
git pull – Сваля най-новите промени от GitHub и ги обединява с вашия локален код.
git remote add origin <url> – Свързва локалното ви хранилище с проект в GitHub.
git fetch – Сваля информация за нови промени от отдалеченото хранилище, без да ги обединява автоматично.

3. Клонове и Разклонения (Branching)
git branch – Списък на всички локални клонове.
git branch <name> – Създава нов клон.
git checkout <name> или git switch <name> – Превключва към друг клон.
git checkout -b <name> – Създава нов клон и веднага превключва към него.
git merge <name> – Обединява промените от посочения клон в текущия.

4. GitHub CLI (gh) – Управление на GitHub функции 
Ако имате инсталиран GitHub CLI, можете да правите следното директно от терминала: 
gh repo create – Създава ново хранилище директно в GitHub.
gh pr create – Създава "Pull Request" (заявка за сливане на код).
gh issue list – Показва списък с отворените проблеми (Issues) по проекта.
